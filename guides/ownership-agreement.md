## 1. RECITALS

## 2. SOURCE CODE OWNERSHIP

### 2.1 Developer's Proprietary Rights

Developer retains 100% ownership of all source code, including but not limited to:

- **Backend Application Code**: All server-side logic, APIs, and business processes
- **Frontend Application Code**: All client-side code, components, and interfaces
- **Database Schemas**: All database structures, tables, relationships, and queries
- **API Implementations**: All application programming interfaces and integration code
- **System Architecture**: Overall design patterns, frameworks, and technical architecture
- **Proprietary Algorithms**: All custom logic, calculations, and processing methods
- **Third-Party Integrations**: All code related to external service connections
- **Documentation**: Technical documentation, code comments, and system specifications

### 2.2 Intellectual Property Protection

Developer maintains all intellectual property rights including:
- Copyright to all original code
- Patent rights (if applicable)
- Trade secrets and proprietary methods
- Technical know-how and expertise

### 2.3 Code Repository & Version Control

- Developer maintains the official code repository
- All code changes are version-controlled and documented
- Client does not receive access to source code repositories
- Code remains proprietary to Developer

---

## 3. CLIENT USAGE RIGHTS

### 3.1 Granted Rights

Client receives full, perpetual, and exclusive usage rights to:

- **Frontend User Interface**: Complete access to all user-facing interfaces
- **User Experience**: Full functionality as described in the functional specification
- **System Functionality**: All features, workflows, and business processes
- **Data Ownership**: Complete ownership of all data entered and generated
- **Report Outputs**: All generated reports, PDFs, and documents
- **System Access**: Unlimited user accounts and system usage

### 3.2 Operational Control

Client has full operational control including:
- Managing user accounts and permissions
- Creating and managing client records
- Configuring business rules and workflows
- Accessing and exporting all data
- Requesting system modifications and enhancements

### 3.3 Data Portability

Client retains rights to:
- Export all data in standard formats
- Create data backups
- Request data migration assistance
- Access historical records indefinitely

## 4. CHANGE MANAGEMENT & VERSION CONTROL

### 4.1 Change Request Protocol

**Authorization Requirements:**
- All change requests must be submitted by the authorized signatory of this agreement
- Requests must be submitted in writing (email acceptable)
- Developer will acknowledge receipt within 2 business days

**Evaluation Process:**
- Developer will evaluate scope, timeline, and technical impact
- Estimated effort and timeline will be provided
- Cost assessment will be provided (if applicable)
- Client approval required before proceeding

### 4.2 Version Control & Documentation

**Developer Responsibilities:**
- Maintain comprehensive version history of all changes
- Document all modifications with version numbers and descriptions
- Provide change logs for client review upon request
- Tag releases with semantic versioning (e.g., 1.0.22)

**Change Categories:**
1. **Critical Security Patches**: Applied immediately, notification within 24 hours
2. **Bug Fixes**: Scheduled and deployed within agreed timeframes
3. **Feature Enhancements**: Evaluated and scheduled based on mutual agreement
4. **Routine Updates**: Scheduled during maintenance windows

### 4.3 Update Notification

Client will receive notification of:
- All planned updates and changes
- Emergency security patches (within 24 hours of deployment)
- New feature releases
- System maintenance schedules

## 5. SYSTEM ACCESSIBILITY & SERVICE LEVELS

### 5.1 Service Commitment

Developer commits to:
- Ensuring reasonable system accessibility and availability
- Monitoring system performance and uptime
- Addressing critical issues promptly
- Maintaining secure hosting infrastructure

### 5.2 Maintenance Windows

**Planned Maintenance:**
- Scheduled during off-peak hours when possible
- Advance notice of at least 48 hours for routine maintenance
- Notice of at least 7 days for major updates
- Coordination with Client for minimal business disruption

**Emergency Maintenance:**
- May be performed without advance notice for critical security issues
- Client notified as soon as reasonably possible
- Post-incident report provided within 72 hours

### 5.3 Uptime Expectations

While Developer strives for maximum availability:
- Best-effort uptime commitment
- Monitoring and alerting systems in place
- Prompt response to reported issues
- Regular system health assessments

## 6. FORCE MAJEURE PROVISIONS

### 6.1 Circumstances Beyond Control

The system may become temporarily inaccessible due to circumstances beyond Developer's reasonable control, including but not limited to:

**Infrastructure Dependencies:**
- Internet service provider outages or network failures
- Hosting provider infrastructure failures
- Content delivery network (CDN) disruptions
- Domain name system (DNS) failures

**Environmental Events:**
- Natural disasters (earthquakes, floods, hurricanes, etc.)
- Extreme weather events
- Fire or other physical damage to infrastructure
- Pandemic or public health emergencies

**Technical Incidents:**
- Distributed denial of service (DDoS) attacks
- Cyber security incidents or breaches
- Hardware failures beyond redundancy capabilities
- Third-party service dependencies

**External Factors:**
- Power outages or utility failures
- Government actions, regulations, or mandates
- Labor disputes or strikes
- War, terrorism, or civil unrest

### 6.2 Force Majeure Response Protocol

**Communication:**
- Client will be promptly notified of any unplanned downtime
- Initial notification within 1 hour of incident detection
- Regular status updates every 4 hours during extended outages
- Final incident report within 72 hours of resolution

**Resolution Efforts:**
- Developer will use commercially reasonable efforts to restore service
- Estimated time to resolution communicated when available
- Alternative solutions explored when feasible
- Priority given to service restoration over non-critical tasks

**Liability Limitation:**
- Developer not liable for delays or failures due to force majeure
- Service level commitments suspended during force majeure events
- Timelines extended by duration of force majeure circumstance

### 7.1 Ongoing Technical Support

**Included Services:**
- Technical support via email and phone during business hours
- Bug fixes and error corrections
- Security updates and patches
- Performance monitoring and optimization
- Regular system health checks
- Database maintenance and optimization

**Response Times:**

**Business Hours:**

- Excluding public holidays
- Emergency contact available for critical issues

### 7.2 Security & Updates

**Security Commitments:**
- Regular security assessments
- Prompt application of critical security patches
- Vulnerability monitoring
- Compliance with security best practices

**Update Schedule:**
- Security patches: As needed (immediate for critical)
- Bug fixes: Monthly or as needed
- Feature updates: Quarterly or as agreed
- Framework updates: As required for security and compatibility

### 7.3 Modification & Enhancement Requests

**Client-Requested Changes:**

- Critical bug fixes: Included in maintenance
- Security updates: Included in maintenance
- Feature enhancements: Evaluated individually
- Major system overhauls: Separate project scope

## 8. DATA OWNERSHIP & PRIVACY

### 8.1 Client Data Ownership

**Complete Ownership:**
- All data entered into the system remains the exclusive property of Client
- Client records, service reports, user information
- Chemical inventory data
- Assignment records and service history
- Generated reports and documents

**Data Rights:**
- Full rights to access, modify, and delete data
- Export data at any time in standard formats
- Request custom data exports or reports
- Migrate data to alternative systems

### 8.2 Data Security & Privacy

**Security Measures:**
- Industry-standard encryption for data transmission and storage
- Secure authentication and access controls
- Regular security audits and assessments
- Compliance with data protection regulations

**Privacy Commitments:**
- Developer will not access Client data except for:
  - Technical support and troubleshooting
  - System maintenance and optimization
  - When explicitly requested by Client
- No sharing of Client data with third parties
- Confidentiality maintained per industry standards

### 8.3 Data Backup & Recovery

**Backup Procedures:**
- Regular automated backups of all system data
- Multiple backup copies maintained
- Secure backup storage with encryption
- Backup retention for disaster recovery

**Recovery Services:**
- Data restoration available upon request
- Point-in-time recovery for recent backups
- Disaster recovery procedures in place
- Regular backup testing and validation

### 8.4 Data Export & Portability

**Export Capabilities:**
- Standard format exports (CSV, Excel, PDF, JSON)
- Complete database exports available
- Custom export formats upon request
- API access for programmatic data retrieval

**Migration Assistance:**
- Developer will assist with data export for migration
- Data provided in formats suitable for import to other systems
- Documentation of data structures provided
- Reasonable assistance during transition period
